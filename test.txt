Author: seelabs <scott.determan@yahoo.com>
Date:   Tue Aug 30 13:50:10 2016 -0400

    Benchmark fetch and insert:
    
    These benchmarks time two operations:
    
    1. The time to insert N values into a data base. The inserted keys and
       values are pseudo-randomly generated. The random number generator is
       always seeded with the same value for each run, so the same values
       are always inserted.
    
    2. The time to fetch M existing values from a data base with N values.
       The order that the keys are fetched are pseudo-randomly generated.
       The random number generator is always seeded with the same value on
       each fun, so the keys are always looked up in the same order.
